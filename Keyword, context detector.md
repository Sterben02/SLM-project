
**Что это:**  
**Keyword/context detector** — детектор, который:

- по **именам переменных** (например, `password`, `api_key`, `secret`)
- по **типу файлов** (например, `.env`, `.key`, `config.yaml`)  определяет, что в коде могут быть секреты.[](https://medium.com/maddevs-io/secrets-513d41eaaf43)

**Простыми словами:**  
Это «умный менеджер», который знает:

- если переменная называется `password` → скорее всего, это секрет
- если файл называется `.env` → там хранятся секреты
- если файл `private.key` → это приватный ключ

**Пример имен переменных-секретов:**

```python
# ❌ Секретные имена
 password = "abc123" 
 api_key = "xyz789" 
 secret_token = "token123" 
 ssh_key = "key_content"`
```
**Пример файлов-секретов:**

|Файл|Тип|
|---|---|
|`.env`|конфигурация с секретами|
|`.key`|ключ|
|`private.pem`|приватный ключ|
|`config.yaml`|конфигурация|

**Плюсы:**

- очень простой
- быстрая работа
- низкая ложноположительность для очевидных случаев

**Минусы:**

- не понимает контекст
- может пропустить секреты с «неочевидными» именами
