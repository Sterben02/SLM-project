# SLM Secret & Insecure Code Scanner

Прототип CLI-сканера исходного кода, сочетающий классические методы (regex, энтропия, ключевые слова) и Small Language Model **Qwen2.5-Coder-1.5B** для поиска секретов и небезопасных фрагментов кода с низким уровнем ложных срабатываний.

---

## Возможности

- **Поиск секретов 5 классов:** API-ключи, access-токены, пароли, приватные ключи, JWT.
- **Обнаружение небезопасного кода 6 классов:** `eval`, `exec`, `shell=True`, SQL-конкатенация, слабые хэши MD5/SHA1, захардкоженные учётные данные.
- **SLM-детектор** на базе **Qwen2.5-Coder-1.5B**, учитывающий контекст (имя переменной, соседние строки, комментарии, тип файла).
- **Объяснения срабатываний** на естественном языке — почему фрагмент считается подозрительным.
- **Отчёты** в форматах JSON и Markdown.
- **Сравнение метрик** baseline-детекторов и SLM-подхода.
- **Глоссарий терминов** — справочник по ключевым понятиям проекта (AppSec, SAST, SLM, NLP и др.).

---

## 🧠 Языковая модель: Qwen2.5-Coder-1.5B

В проекте используется модель [**Qwen2.5-Coder-1.5B-Instruct**](https://huggingface.co/Qwen/Qwen2.5-Coder-1.5B-Instruct) от команды Qwen (Alibaba).

### Почему именно эта модель

| Критерий | Обоснование |
|---|---|
| **Специализация на коде** | Обучена на 5.5 трлн токенов, включая большой объём исходного кода на 92+ языках программирования. |
| **Размер 1.5B параметров** | Оптимальный баланс качества и скорости. Помещается в RAM обычного ноутбука. |
| **Контекстное окно 32K токенов** | Позволяет передавать не только подозрительную строку, но и 10-20 строк контекста до и после. |
| **Поддержка русского языка** | Мультиязычная модель — важно для объяснений на русском. |
| **Локальный inference** | Код не покидает машину пользователя — критично при поиске секретов. |
| **Лицензия Apache 2.0** | Свободное использование в исследовательских и коммерческих целях. |
| **Instruction-tuned** | Версия `-Instruct` оптимизирована для следования инструкциям — идеально для классификации через промпты. |

### Альтернативы, которые рассматривались

| Модель | Причина, почему не выбрана |
|---|---|
| Phi-3-mini (3.8B) | В 2.5 раза больше, медленнее, не даёт пропорционального прироста качества |
| SmolLM-1.7B | Менее специализирована на коде |
| CodeLlama-7B | Слишком большая для CPU, требует GPU |
| Qwen2.5-1.5B (базовая) | Не instruction-tuned, хуже следует промптам |

### Характеристики

- **Параметры:** 1.5 млрд
- **Архитектура:** Transformer (decoder-only)
- **Контекст:** до 32 768 токенов
- **Языки:** 92+ языков программирования + естественные языки
- **Формат:** HuggingFace `transformers` (safetensors)
- **Размер на диске:** ~3.2 GB (fp16), ~0.9 GB (int4)
- **Требования к RAM:** ~4 GB (fp16), ~2 GB (int4)
