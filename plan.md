# План работы над проектом SLM
## Этап 1: Изучение терминологии
Для начала были изучены термины и определения различных понятий, необходимых для понимания темы и дальнейших шагов.
### Перечень изученных терминов:
- [Секреты в коде](glossary/Secrets_in_code.md) — определение и примеры секретов
- [SLM](glossary/SLM.md) — Statistical Language Model
- [AppSec](glossary/AppSec.md) — Application Security
- [SAST](glossary/SAST.md) — Static Application Security Testing
- [NLP; NER](glossary/NLP-NER.md) — Natural Language Processing & Named Entity Recognition
- [Небезопасный фрагмент кода](glossary/Unsafe_code_fragment.md) — определение небезопасного кода
- [Typer](glossary/Typer.md) — библиотека для создания CLI
- [Argparse](glossary/Argparse.md) — библиотека для парсинга аргументов командной строки
- [Regex-based detector](glossary/Regex-based-detector.md) — детектор на основе регулярных выражений
- [Entropy-based detector](glossary/Entropy-based-detector.md) — детектор на основе энтропии
- [Keyword, context detector](glossary/Keyword-context-detector.md) — контекстный детектор ключевых слов
- [Ошибки 1 и 2 рода](glossary/Errors_of_the_1st_and_2nd_kind.md) — основные понятия статистики
### Источник информации
Для поиска информации по данным терминам и определениям была использована нейросеть **Perplexity**.
