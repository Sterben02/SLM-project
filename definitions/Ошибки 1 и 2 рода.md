**Что это:**  
В статистике и машинном обучении:

- **Ошибка первого рода** = **False Positive** (ложноположительная) — система сказал «да», но это «нет»
- **Ошибка второго рода** = **False Negative** (ложноотрицательная) — система сказал «нет», но это «да»

**Простыми словами:**  
Пример с детектором секретов:

| Ситуация              | Ошибка 1-го рода                     | Ошибка 2-го рода                    |
| --------------------- | ------------------------------------ | ----------------------------------- |
| Детектор нашёл секрет | ✅ Секрет есть → ложноположительный   | ❌ Секрет нет → ложный отрицательный |
| Детектор не нашёл     | ❌ Секрет есть → пропустил уязвимость | ✅ Секрет нет → всё правильно        |

**Таблица ошибок:**

| Секрет **есть**       | Секрет **нет**                 |                                |
| --------------------- | ------------------------------ | ------------------------------ |
| Детектор **нашёл**    | ✅ True Positive                | ❌ **False Positive** (1-й род) |
| Детектор **не нашёл** | ❌ **False Negative** (2-й род) | ✅ True Negative                |

**Примеры:**

**1-й род (False Positive):**

```text
Детектор сказал: "В коде есть пароль!" Но на самом деле: это просто слово "password" в комментарии, не секрет.
```

**2-й род (False Negative):**

```text
Детектор сказал: "В коде нет секретов." Но на самом деле: пароль скрыт в переменной `x = "abc123"`.
```

**Какая ошибка хуже?**

|Область|1-й род|2-й род|
|---|---|---|
|Детектор уязвимостей|Ложное предупреждение = потеря времени|**Пропущена уязвимость = взлом** ❌|
|Медицинский тест|Ложный диагноз = лишний осмотр|**Пропущена болезнь = смерть** ❌|
|Спам-фильтр|Важное письмо в спам|**Спам в почте**|

В безопасности **2-й род (False Negative)** обычно хуже, потому что пропускается реальная угроза.
