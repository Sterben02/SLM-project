**Что это:**  
Секреты — это чувствительные данные, которые нельзя хранить в открытом виде в коде: пароли, API-ключи, SSH-ключи, строки подключения к базам данных, цифровые сертификаты, номера карт и т.п.

**Простыми словами:**  
Это как «подписи» или «ключи доступа» к важным системам. Если разработчик кладёт пароль от базы данных прямо в код программы, и этот код попадает в интернет (например, в GitHub), любой может получить доступ к вашей базе.

**Пример секрета:**

```python

# ❌ НЕПРАВИЛЬНО — секрет в коде 
password = "my_super_secret_123" db_connection = "mysql://user:my_super_secret_123@localhost/db"`
```

```python

# ✅ ПРАВИЛЬНО — через переменные окружения 
import os 
password = os.getenv("DB_PASSWORD")  # пароль хранится в .env файле`
```

**Почему это опасно:**  
Утечка секретов = возможность злоумышленникам получить доступ к вашим системам, данным, инфраструктуре.

**Как правильно работать:**

- never хранить секреты в коде
- использовать системы управления секретами: HashiCorp Vault, Azure Key Vault
- настроить автоматическую проверку на секреты: GitLeaks, TruffleHog, DeepSecrets
